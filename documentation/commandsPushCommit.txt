📋 Твой правильный алгоритм (с нуля)
bash
# 1. Инициализируем репозиторий в папке
git init

# 2. Связываем с GitHub
git remote add origin https://github.com/ruslan9814/testProject.git

# 3. Добавляем все файлы
git add .

# 4. Делаем первый коммит
git commit -m "v1"

# 5. Отправляем на GitHub
git push -u origin main
🔍 Что происходит на каждом шаге
Шаг	Команда	Что делает
1	git init	Создает .git папку — теперь Git следит за папкой
2	git remote add origin ...	Говорит Git-у: "Вот адрес твоего дома на GitHub"
3	git add .	Кладёт ВСЕ файлы в корзину (готовит к отправке)
4	git commit -m "v1"	Делает снимок (сохраняет версию) с названием "v1"
5	git push -u origin main	Отправляет ВСЕ коммиты на GitHub и запоминает связь
💡 Важные нюансы
После первого push -u можно писать просто git push
bash
# Первый раз (с -u)
git push -u origin main

# Все следующие разы (уже без -u)
git push
Почему: -u (upstream) устанавливает связь. Git запоминает, что твоя локальная main связана с origin/main.

Если файлов нет — git add . ничего не добавит
bash
git add .
git commit -m "v1"
# Ошибка: nothing to commit
Решение: Создай хотя бы один файл:

bash
echo "Hello" > readme.txt
git add .
git commit -m "v1"
Если на GitHub уже есть коммиты (а у тебя нет)
bash
git push -u origin main
# Ошибка: rejected (non-fast-forward)
Решение: Сначала скачай чужие изменения:

bash
git pull origin main
# Теперь объедини
git push

📊 Твоя типовая рутина (ежедневная)
После того как ты настроил проект (сделал init, remote add, первый push):

bash
# 1. Убедись, что ты на правильной ветке
git branch

# 2. Проверь статус
git status

# 3. Добавь изменения

# 4. Сохрани

# 5. Отправь на GitHub
git push
